01 / Naming

The initiative needs its own public name.

Aurelius Dynamic stays the parent company. The open initiative sitting on top of it should read like a commons, not a product launch. Here is the strategy, then six candidates in two families to react to.

Naming strategy

A public name, parented by Aurelius.

The name should sound like infrastructure, hold several projects under one roof, and stay quiet about the K1 mission. Products keep their own names and live inside it.

  • Holds many. One roof: app, research, hardware.
  • Open by default. Reads as a commons, not a launch.
  • Ownable. Clean TM, .org / .science / .dev, GitHub org.
  • Quiet on K1. The mission is substrate, not the label.
  • Pairs. Works as "[name], an Aurelius initiative."
How it nests
